
**流程图（纯文本查询）：**
```
[用户文本查询] -> [文本编码器] -> [文本向量] -> [检索] -> [多模态上下文] -> [LLM生成] -> [文本回答 (+图片引用)]
```
**流程图（图文查询）：**
```
                [用户图片] -> [多模态编码器] -> [联合向量] -> [检索] -> [多模态上下文] -> [MLLM生成] -> [图文并茂的回答]
                [用户文本] -/ 
                    (或直接使用多模态编码器处理图文对)
```

---

### 核心区别总结

| 方面 | 纯文本提问 | 文本 + 图片提问 |
| :--- | :--- | :--- |
| **查询编码** | 使用**文本编码器**生成**文本向量**。 | 使用**多模态编码器**（如CLIP）生成**联合向量**，或将图文向量融合。 |
| **检索依据** | 查询的**文本语义**。 | 查询的**图文联合语义**。例如，图片的视觉特征与文本的抽象意图相结合。 |
| **检索精度** | 对于纯文本概念和描述性需求效果好。 | 当查询意图严重依赖视觉信息（如风格、形状、颜色、具体物体）时，精度远高于纯文本检索。 |
| **生成模型** | 主要使用**纯文本LLM**（如GPT-3.5-Turbo, Llama）。 | 必须使用**多模态大模型**（如GPT-4V, Gemini）来理解用户上传的图片。 |
| **系统复杂度** | 相对简单，是传统RAG的直接延伸。 | 更复杂，需要集成多模态编码、多模态向量库和多模态生成模型。 |
| **应用场景** | 事实问答、文档摘要、基于文本的推荐。 | 以图搜图、视觉问答、设计灵感查找、商品搜索（拍照搜索）、医疗影像分析等。 |
